Clone and Own

  • Simple: ctrl+c ctrl+v
  • new Project: copy a similar one
  • project templates are essentially clones
  • not always bad, but smelly
  • creates bugfixing, upgrading problems

Some Alternatives

  • OO Inheritance - small scale
  • Libraries - require integration
  • Components - very specific
  • Preprocessors - ifdef hell
  • Frameworks/Plugin Systems - high preplanning effort, runtime penalty
  • negative Variability: hide functionality on demand

Feature Oriented Programming

What is a feature?

  • satisfy a stakeholders requirement
  • implement and encapsulate a design decision
  • provide an increment of functionality
  • present a potential configuration option

Composing CSS

CSS composes quite nicely

BASE feature: 
.mybutton {
    background: blue;
}
CUSTOMER_A_CI feature: 
.mybutton {
    background: green;
    border-radius: 5px;
}
 CUSTOMER_A_CI refines the mybutton class:
  • refines its background
  • introduces a border radius

Composing JAVA

BASE feature: 
class A {
    float foo() {
        return 3.14;
    }
}
FEATURE_A feature: 
class A {
    float foo() {
        return original() + 2.78;
    }
}

Benefits

Technical Benefits

  • feature model can be analyzed
  • dedicated abstraction for variability management
  • concept is easy to communicate
  • directly maps to requirements
  • feature tracebility (code of a feature is no longer scattered across the codebase)

Benefits

Example organizational benefits include:

  • Improved productivity by as much as 10x
  • Increased quality by as much as 10x
  • Decreased cost by as much as 60%
  • Decreased labor needs by as much as 87%
  • Decreased time to market (to field, to launch) by as much as 98%
  • Ability to move into new markets in months, not years
(acc. to http://www.sei.cmu.edu/productlines/)
